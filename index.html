Remote Code Publisher

Developed means to display source code files as web pages with embedded child links. Each link refers to a code file that the displayed code file depends on.

Developed a remote code publisher, local client, and communication channel that supports client access to the publisher from any internet enabled processor.

The communication channel made use of sockets and supported HTTP style request/response transactions, One-way communication allowing asynchronous messaging between any two endpoints that are capable of listening for connection requests and connecting to a remote listener. Transmission of byte streams were set up with one or more negotiation messages followed by transmission of a stream of bytes of specified stream size.

The Remote Code Publisher supported publishing web pages that were small wrappers around C++ source code files, supported building dependency relationships between code files saved in specific repository folders, supported HTML file creation for all the files in a specified repository folder, including linking information that displays dependency relationships and also supported deleting stored files, as requested by a local client.

The Client of the Source Code file was provided using a Graphical User Interface (GUI).

Technologies Used: C++, Windows Presentation Foundation (WPF), C++/Cli, HTML/CSS, JS.

Remote Test Harness

This project is an Implementation for accessing and using a remote Test Harness server from multiple concurrent clients. The Test Harness retrieves test code from a Repository server.

One or more client(s) concurrently supply the Test Harness with Test Requests. The Test Harness requests test drivers and code to test, as cited in a Test Request, from the Repository.

One or more client(s) concurrently extract Test Results and logs by enqueuing requests and displaying Test Harness and/or Repository replies when they arrive. The TestHarness, Repository, and Clients are all distinct projects that compile to separate executables.

All communication between these processes are based on message-passing Windows Communication Foundation (WCF) channels. Client activities are defined by user actions in a Windows Presentation Foundation (WPF) user interface.

Technologies Used: C#, Windows Communication foundation (WCF), Windows Presentation foundation (WPF)
